# pip install dvc
4. Now we do "dvc init" (creates .dvcignore, .dvc)

5. Now do "mkdir S3" (creates a new S3 directory) [We did it because, we want to give address where saved the data-versioning which is done by DVC]

6. Now we do "dvc remote add -d myremote S3"

7. Next "dvc add data/" [We want to data versioning through DVC With only Data Folder's files or folders]
   Now it will ask to do: ("git rm -r --cached 'data'" and "git commit -m "stop tracking data"")
   Because initially we were tracking data/ folder from git so now we remove it for DVC to handle.

8. Again we do "dvc add data/" (creates data.dvc) then "git add .gitignore data.dvc"

    [Run Commands: git add . , git commit -m "DVC initiated with data version1", git push origin main, git status, dvc status]

9. Now - "dvc commit" and then "dvc push" [Ye karne se, jo data data folder ke ander hai usko utha ke S3 me dal dega with a md5 hash, aur jab jab hum apne data me changes karenge utne baar ye dono file (S3 ke ander wala) ke datas change honge (like md5 hash, data itself)]

9. Do a git add-commit-push to mark this stage as first version of data. [Mane git add . ....]
